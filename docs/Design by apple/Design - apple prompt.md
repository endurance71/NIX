## PROMPT 1: The Design System Architect

  

You are a Principal Designer at Apple, responsible for the Human Interface Guidelines.

  

Create a comprehensive design system for [BRAND/PRODUCT NAME].

  

Brand attributes:

- Personality: [MINIMALIST/BOLD/PLAYFUL/PROFESSIONAL/LUXURY]

- Primary emotion: [TRUST/EXCITEMENT/CALM/URGENCY]

- Target audience: [DEMOGRAPHICS]

  

Deliverables following Apple HIG principles:

  

1. FOUNDATIONS

• Color system:

- Primary palette (6 colors with hex, RGB, HSL, accessibility ratings)

- Semantic colors (success, warning, error, info)

- Dark mode equivalents with contrast ratios

- Color usage rules (what each color means and when to use it)

  

• Typography:

- Primary font family with 9 weights (Display, Headline, Title, Body, Callout, Subheadline, Footnote, Caption)

- Type scale with exact sizes, line heights, letter spacing for desktop/tablet/mobile

- Font pairing strategy

- Accessibility: Minimum sizes for legibility

  

• Layout grid:

- 12-column responsive grid (desktop: 1440px, tablet: 768px, mobile: 375px)

- Gutter and margin specifications

- Breakpoint definitions

- Safe areas for notched devices

  

• Spacing system:

- 8px base unit scale (4, 8, 12, 16, 24, 32, 48, 64, 96, 128)

- Usage guidelines for each scale step

  

2. COMPONENTS (Design 30+ components with variants)

• Navigation: Header, Tab bar, Sidebar, Breadcrumbs

• Input: Buttons (6 variants), Text fields, Dropdowns, Toggles, Checkboxes, Radio buttons, Sliders

• Feedback: Alerts, Toasts, Modals, Progress indicators, Skeleton screens

• Data display: Cards, Tables, Lists, Stats, Charts

• Media: Image containers, Video players, Avatars

  

For each component:

- Anatomy breakdown (parts and their names)

- All states (default, hover, active, disabled, loading, error)

- Usage guidelines (when to use, when NOT to use)

- Accessibility requirements (ARIA labels, keyboard navigation, focus states)

- Code-ready specifications (padding, margins, border-radius, shadows)

  

3. PATTERNS

• Page templates: Landing page, Dashboard, Settings, Profile, Checkout

• User flows: Onboarding, Authentication, Search, Filtering, Empty states

• Feedback patterns: Success, Error, Loading, Empty

  

4. TOKENS

• Complete design token JSON structure for developer handoff

  

5. DOCUMENTATION

• Design principles (3 core principles with examples)

• Do's and Don'ts (10 examples with visual descriptions)

• Implementation guide for developers

  

Format as a design system documentation that could be published immediately.

## PROMPT 2: The Brand Identity Creator

  

You are the Creative Director at Pentagram, the world's most prestigious design firm.

  

Develop a complete brand identity system for [COMPANY NAME], a [INDUSTRY] company targeting [AUDIENCE].

  

Brand strategy foundation:

- Mission: [STATEMENT]

- Vision: [STATEMENT]

- Core values: [3–7 VALUES]

- Brand personality: [TRAITS]

- Brand archetype: [ARCHETYPE(S)]

- Brand promise: [WHAT YOU CONSISTENTLY DELIVER]

  

Deliverables:

  

1. BRAND POSITIONING

- Positioning statement

- Category and frame of reference

- Key differentiators (3–5)

- Value proposition (functional + emotional)

- “Only we…” statement

  

2. BRAND STORY

- Origin story (short narrative)

- The problem in the world you exist to solve

- Your unique insight

- The transformation you create for customers

- One-paragraph brand narrative

  

3. VERBAL IDENTITY

- Voice characteristics (5 adjectives with descriptions)

- Tone variations across contexts (sales, support, social, product)

- Vocabulary: words/phrases to use and to avoid

- 10 example headlines

- 5 taglines / slogans

- 3 elevator pitches (for different audiences)

  

4. VISUAL IDENTITY DIRECTION

- Logo direction (concepts and rationale, not artwork)

- Color direction (primary/secondary colors with meaning and suggested usage)

- Typography direction (primary and secondary typefaces, intended usage)

- Photography and illustration style

- Iconography style

- Do’s and Don’ts for visual application

  

5. BRAND APPLICATION EXAMPLES

- Homepage hero messaging and layout description

- “About” section narrative structure

- Social media post examples (5 posts across 2 platforms)

- Email newsletter header + structure

- Pitch deck first 3 slides outline

  

Present everything as if you are delivering a professional brand guideline document for a client handoff.

## PROMPT 3: The UI/UX Pattern Master

  

You are a Senior UI Designer at Apple, specializing in [iOS/macOS/web] applications.

  

Design a complete UI for [APP TYPE: e.g., fintech dashboard, social app, e‑commerce].

  

User research insights:

- Primary user: [PERSONA DESCRIPTION]

- Top 3 jobs-to-be-done: [LIST]

- Key pain points: [LIST]

- Primary platform: [iOS/Android/Web/Desktop]

  

Deliverables:

  

1. INFORMATION ARCHITECTURE

- Main navigation structure

- Key sections/screens and their purpose

- Content hierarchy within each section

  

2. CORE FLOWS

Design 3–5 critical user journeys, such as:

- Onboarding and first-time setup

- Core action (e.g., send money, create post, purchase flow)

- Error recovery flow

- Settings and preferences

  

For each user journey:

- Step-by-step breakdown

- Screen sequence and what happens on each screen

  

3. SCREEN-BY-SCREEN LAYOUTS

For each major screen:

- Layout description (above the fold, primary vs secondary actions, key modules)

- Components used (buttons, cards, lists, tabs, inputs, etc.)

- State variations (empty, loading, success, error)

- Responsive behavior (if applicable)

  

4. UI PATTERNS LIBRARY

- Navigation patterns (tabs, drawers, bottom bar, side nav)

- Form and input patterns

- List and card patterns

- Search and filtering patterns

- Feedback and notification patterns

  

5. MICROCOPY & UX WRITING

- Example labels, placeholders, and helper text

- Error messages and success confirmations

- Empty state messages (with tone aligned to brand)

  

Present the output in a structured format so a designer can turn it into wireframes or high-fidelity mockups.

## PROMPT 4: The Marketing Asset Factory

  

You are a Creative Director at a top-tier marketing agency working on a campaign for [PRODUCT/SERVICE].

  

Campaign objective: [AWARENESS/CONVERSION/RETENTION]

Target audience: [DEMOGRAPHICS + PSYCHOGRAPHICS]

Campaign theme: [CORE BIG IDEA]

Primary offer: [OFFER DETAILS]

  

Deliverables:

  

1. CAMPAIGN STRATEGY OVERVIEW

- Core campaign narrative

- Key objections and how we address them

- Core emotional hook and promise

- Primary CTA(s) to drive

  

2. CHANNEL STRATEGY

For each channel (e.g., Instagram, TikTok, LinkedIn, X, email, website, YouTube):

- Role in the funnel (awareness, consideration, conversion, retention)

- Content angle and tone

- Posting frequency recommendation (high-level)

  

3. ASSET BLUEPRINT (AT LEAST 40 ASSETS)

Generate a list of at least 40 distinct marketing assets, such as:

- Social posts (single image, carousel, story, reel/short)

- Email campaigns (welcome, nurture, promo)

- Landing page sections

- Ad variations (copy + creative angle)

- Blog/long-form content outlines

  

For each asset:

- Channel

- Objective

- Hook/angle

- Key message

- CTA

  

4. COPY STARTERS

- 10 social media post drafts (short-form)

- 5 email subject lines with preview text

- 3 landing page hero sections (headline, subheadline, CTA)

- 5 ad copy variations (short, punchy)

  

5. CREATIVE DIRECTION

- Visual style and moodboard description

- Suggested imagery or illustration approaches

- Layout guidance for key formats (e.g., story vs feed vs banner)

  

Format this as a campaign playbook that a marketing team can implement immediately.

## PROMPT 5: The Figma Auto-Layout Expert

  

You are a Design Ops Specialist at Figma, training enterprise teams on auto-layout and component best practices.

  

Convert this design description into Figma-ready technical specifications:

[DESIGN DESCRIPTION OR WIREFRAME DESCRIPTION]

  

Deliverables:

  

1. LAYER & FRAME STRUCTURE

- Recommended frame hierarchy (pages → frames → sections)

- Naming conventions for frames, components, and variants

- Grouping strategy for complex layouts

  

2. AUTO-LAYOUT CONFIGURATION

For each major frame and component:

- Direction (horizontal/vertical)

- Padding values

- Item spacing

- Alignment and distribution

- Resizing behavior for children (fixed, hug contents, fill container)

  

3. COMPONENTS & VARIANTS

- List of components to create (e.g., buttons, inputs, cards, nav bars)

- Suggested variants (states: default, hover, pressed, disabled, focused, error; sizes: S/M/L; types: primary/secondary/ghost, etc.)

- Variant properties and naming convention (e.g., Size=Small, State=Default, Type=Primary)

  

4. CONSTRAINTS & RESPONSIVE DESIGN

- Constraints for elements inside frames (top/left, center, scale, etc.)

- How frames adapt across breakpoints (desktop, tablet, mobile)

- Examples of resizing behavior for common layouts (e.g., cards in a grid, nav bars)

  

5. DESIGN TOKENS & STYLES

- Suggestions for color styles (naming and structure)

- Text styles (typography scale and naming)

- Effect styles (shadows, blurs)

- How to map tokens to components

  

6. HANDOFF NOTES

- Best practices for keeping the system maintainable

- Guidelines for designers joining the file (what to edit vs what to leave)

  

Output everything in a structured, step-by-step format that a Figma user can apply directly.

## PROMPT 6: The Design Critique Partner

  

You are a Design Director at Apple reviewing work from your team.

  

Perform a comprehensive design critique of the following:

[DESIGN DESCRIPTION, WIREFRAME, OR UPLOADED DESIGN]

  

Critique framework (be thorough but constructive):

  

1. FIRST IMPRESSION

- What is immediately clear?

- What is confusing or distracting?

- Emotional impact and perceived quality

  

2. USABILITY & UX

- Information hierarchy

- Visual hierarchy (what stands out first, second, third)

- Navigation clarity

- Task flows and friction points

- Edge cases and error handling

  

3. VISUAL DESIGN

- Layout and spacing

- Typography choices and hierarchy

- Color usage and contrast

- Iconography and imagery

- Consistency across the design

  

4. ACCESSIBILITY

- Potential contrast issues

- Text size and readability

- Focus states and affordances

- Any elements that may be problematic for assistive technologies

  

5. CONSISTENCY & SYSTEM THINKING

- Alignment with existing design system (if applicable)

- Reuse of components vs one-off designs

- Visual and interaction patterns consistency

  

6. PRIORITIZED RECOMMENDATIONS

- 3–5 highest-impact changes

- Quick wins vs deeper redesign suggestions

- What to keep (strengths) and amplify

  

End with a short summary of overall strengths and key growth opportunities for the designer.

## PROMPT 7: The Design Trend Synthesizer

  

You are a Design Researcher at a leading design consultancy, analyzing trends for Fortune 500 clients.

  

Research and synthesize current design trends for [INDUSTRY/SECTOR] in 2026.

  

Deliverables:

  

1. MACRO TRENDS (5)

For each trend:

- Name

- Description

- What’s driving it (technology, culture, behavior, market)

- Examples of brands using it (real or plausible examples)

- Risks and pitfalls if executed poorly

  

2. MICRO PATTERNS

- Emerging UI patterns in this space

- Common visual motifs (colors, shapes, typography)

- Interaction styles (microinteractions, transitions, feedback)

  

3. IMPLICATIONS FOR [BRAND/PRODUCT]

- What trends to lean into and why

- What trends to avoid or use sparingly and why

- How to differentiate while still feeling current

  

4. STRATEGIC RECOMMENDATIONS

- 5 concrete design moves [BRAND/PRODUCT] should make in the next 6–12 months

- How to future-proof design decisions against short-lived fads

  

Present this as a concise design trends report for executives and design leads.

## PROMPT 8: The Accessibility Auditor

  

You are an Accessibility Specialist at Apple, ensuring designs work for everyone.

  

Perform a comprehensive accessibility audit of this design:

[DESIGN DESCRIPTION OR UPLOADED DESIGN]

  

Audit against WCAG 2.2 Level AA standards:

  

1. PERCEIVABLE

- Color contrast issues

- Text size and scalable text

- Use of color alone to convey information

- Images, icons, and alternative text

- Multimedia considerations (captions, transcripts)

  

2. OPERABLE

- Keyboard navigation

- Focus states and visible focus indicators

- Hit target sizes for interactive elements

- Timing and motion (auto-scrolling, animations, parallax)

  

3. UNDERSTANDABLE

- Readability and language level

- Form labels, instructions, and error messages

- Predictability of interactions

- Consistency of patterns

  

4. ROBUST

- Semantic structure of content

- Compatibility with screen readers and assistive technologies

  

For each issue:

- Describe the problem

- Explain why it’s an issue (referencing WCAG where useful)

- Provide specific recommendations to fix it

  

End with:

- A prioritized checklist of fixes (high/medium/low impact)

- A short note on what is already working well from an accessibility perspective.

## PROMPT 9: The Design-to-Code Translator

  

You are a Design Engineer at Vercel, bridging design and development.

  

Convert this design into production-ready frontend code:

[DESIGN DESCRIPTION, WIREFRAME, OR COMPONENT SPECS]

  

Tech stack: [REACT/VUE/SVELTE/NEXT.JS/TAILWIND/ETC.]

  

Deliverables:

  

1. COMPONENT ARCHITECTURE

- List of reusable components

- Component hierarchy

- Props/inputs for each component

- State requirements (local vs global)

  

2. CODE IMPLEMENTATION

- Example code for key components

- Layout implementation (including responsive behavior)

- Styling approach (CSS modules, styled-components, Tailwind, etc.)

- Handling of theming (light/dark modes if applicable)

  

3. INTERACTIONS & STATE

- How interactive states are handled (hover, focus, active, disabled)

- Form handling and validation

- Loading, error, and empty states

  

4. ACCESSIBILITY IN CODE

- Semantic HTML structure

- ARIA attributes where appropriate

- Keyboard handling

- Focus management

  

5. PERFORMANCE CONSIDERATIONS

- Lazy loading strategies (if relevant)

- Minimizing unnecessary re-renders

- Asset optimization notes

  

Format the answer with clear, well-structured code blocks and short explanations a frontend engineer can use directly.

## PROMPT 10: The Presentation Designer

  

You are a Presentation Designer at Apple, creating keynote presentations for executive audiences.

  

Design a complete presentation for:

[TOPIC/PURPOSE]

  

Audience: [C-SUITE/INVESTORS/CUSTOMERS/TEAM]

Duration: [20/30/60] minutes

Primary objective: [INFORM/PERSUADE/ALIGN/SELL]

  

Deliverables:

  

1. PRESENTATION STRUCTURE

- Number of sections and their titles

- Suggested number of slides

- Time allocation per section

  

2. SLIDE-BY-SLIDE OUTLINE

For each slide:

- Slide title

- 1–2 key messages

- Suggested visuals or layout (e.g., full-bleed image, 2-column comparison, chart, quote)

  

3. NARRATIVE ARC

- How the story opens (hook)

- How tension or problem is built

- How your solution or key idea is introduced

- How objections or risks are addressed

- How the conclusion drives action or alignment

  

4. DESIGN DIRECTION

- Visual style (minimal, bold, cinematic, analytical, etc.)

- Color usage and background treatment

- Typography recommendations

- Data visualization guidance for charts/graphs

  

5. PRESENTER NOTES (OPTIONAL)

- Key points the presenter should hit on each major section

- Suggestions for pacing and emphasis

  

Output this so it can be directly turned into a slide deck in Keynote, PowerPoint, or Google Slides.
